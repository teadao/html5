рекомендуется всегда включать title атрибут для iframe. Это используется программами чтения с экрана для чтения содержимого iframe.
Используйте height и width атрибуты чтобы указать размер фрейма. По умолчанию высота и ширина указаны в пикселях. Или вы можете добавить style атрибут и использовать CSS свойства height и width.
По умолчанию iframe имеет рамку вокруг себя. Чтобы удалить границу, добавьте атрибут style и используйте свойство CSS border:none. С помощью CSS вы также можете изменить размер, стиль и цвет границы iframe.

iframe можно использовать в качестве целевого фрейма для ссылки.

HTML JavaScript

JavaScript делает HTML-страницы более динамичными и интерактивными.
Тег HTML script используется для определения клиентского сценария (JavaScript).
Элемент script либо содержит утверждения сценария, или он указывает на внешний файл сценария через src атрибут.
Обычно JavaScript используется для обработки изображений, проверки формы и динамического изменения содержимого.
Для выбора элемента HTML в JavaScript чаще всего используется этот метод document.getElementById().

JavaScript может изменить содержимое HTML-элемента.
JavaScript может изменить стиль элемента HTML.

Здесь JavaScript изменяет значение атрибута src (исходный адрес) изображения.

Ссылки на файлы в HTML

Ссылка на файл описывает расположение файла в структуре папок веб-сайта или указывает путь на внешние файлы, например:
  • веб-страница
  • Изображения
  • Таблицы стилей
  • JavaScripts
  • и т.д.

примеры относительных ссылок:

  • img src = "картинка.jpg" Файл "картинка.jpg" находится в той же папке, что и текущая страница.
  • img src = "папка/картинка.jpg"> Файл "картинка.jpg" находится в папке изображений в текущей папке.
  • img src = "/ папка / картинка.jpg" Файл "картинка.jpg" находится в папке изображений в корне текущего веб-сайта.
  • img src=".../папка /картинка.jpg" Файл "picture.jpg находится в папке на один уровень выше текущей папки.

примеры абсолютных ссылок:

  • img src="https://vjoy.cc/wp-content/uploads/2019/10/2-7.jpg"
Лучше всего использовать относительные пути к файлам (если возможно). При использовании относительных путей к файлам ваши веб-страницы не будут привязаны к вашему текущему базовому URL. Все ссылки будут работать на вашем собственном компьютере (localhost), а также в вашем текущем общедоступном домене и ваших будущих общедоступных доменах.

HTML элемент Head

HTML элемент head является контейнером для следующих элементов: title, style, meta, link, script, и base.

Элемент head является контейнером для метаданных (данные о данных) и помещается между тегами html и body.
Метаданные HTML - это данные о документе HTML которые обычно определяют заголовок документа, набор символов, стили, сценарии и другую метаинформацию. Метаданные не отображаются.

HTML Элемент title

Элемент title определяет заголовок документа. Заголовок должен быть только текстовым, и он отображается в строке заголовка браузера или на вкладке страницы.
Тег title обязателен в HTML-документах! Содержание заголовка страницы очень важно для поисковой оптимизации (SEO)! Заголовок страницы используется алгоритмами поисковой системы для определения порядка при перечислении страниц в результатах поиска.

Элемент title:

  • определяет заголовок на панели инструментов браузера
  • предоставляет заголовок для страницы, когда она добавляется в избранное
  • отображает заголовок страницы в результатах поиска

Элемент style используется для определения стиля одной HTML страницы

Элемент link определяет отношения между текущим документом и внешним ресурсом. Тег чаще всего используется для ссылки на внешние таблицы стилей


HTML Элемент meta

Элемент meta обычно используется для указания кодировки символов, описания страницы, ключевых слова, автора документа, а также настроек области просмотра.
Метаданные не будут отображаться на странице, они используются браузерами (как отображать контент или как часто перезагружать страницу), поисковыми системами (ключевые слова) и другими веб-службами.

Примеры meta:

  • meta charset="UTF-8" - указание кодировки символов
  • meta name="description" content="как написать сайт" - Определяет описание вашей веб-страницы.
  • meta name="keywords" content="как, HTML, Java скрипт" - Определите ключевые слова для поисковых систем.
  • meta name="author" content="Albert Guard" - Определяет автора страницы.
  • meta http-equiv="refresh" content="30" - Обновляет документ каждые 30 секунд.
  • meta name="viewport" content="width=device-width, initial-scale=1.0" - Настройка области просмотра, чтобы ваш сайт выглядел хорошо на всех устройствах.
    • Настройка области просмотра:Область просмотра - это видимая пользователем область веб-страницы. Это зависит от устройства - на мобильном телефоне он будет меньше, чем на экране компьютера. Вы должны включить следующие элемены meta на все свои веб-страницы: meta name="viewport" content="width=device-width, initial-scale=1.0".
      Это дает браузеру инструкции по управлению размерами и масштабированием страницы. часть width=device-width устанавливает ширину страницы в соответствии с шириной экрана устройства (которая будет варьироваться в зависимости от устройства). Часть initial-scale=1.0 устанавливает начальный уровень масштабирования при первой загрузке страницы в браузере.

HTML Элемент base

Элемент base определяет базовый URL и / или цель для всех относительных URL - адресов в странице.
Тег base должен иметь или HREF или целевой атрибут настоящего, или оба.
В документе может быть только один base элемент!
пример:

head
base href="https://www.w3schools.com/" target="_blank">
/head
мы указали только относительный адрес изображения. Поскольку мы указали базовый URL-адрес в разделе head, браузер будет искать изображение по адресу «https://www.w3schools.com/images/stickman.gif».
body
img src="images/stickman.gif" width="24" height="39" alt="Stickman">
a href="tags/tag_base.asp"/a
/body
ссылка открывается в новом окне, даже если у неё нет атрибута target = "_ blank". Это связано с тем, что целевой атрибут базового элемента установлен на «_blank».
